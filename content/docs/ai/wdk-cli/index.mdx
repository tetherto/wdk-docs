WDK CLI is a local wallet command line interface for humans, scripts, and AI agents. It wraps WDK wallet modules in three local processes:

| Process | What it does |
| --- | --- |
| `wdk` | Parses commands, prints human-readable or JSON output, and sends wallet requests to the daemon |
| `wdk-daemon` | Holds unlocked WDK wallet instances in memory, derives addresses, signs transactions, and locks wallets after their TTL |
| `wdk-mcp` | Exposes wallet operations as Model Context Protocol tools for MCP-compatible AI clients |

<Callout type="warn">
**Draft source install only** - The WDK CLI implementation is currently under review in [tetherto/wdk-cli#1](https://github.com/tetherto/wdk-cli/pull/1). Use the source-install flow until the module is merged and a non-placeholder package is published.
</Callout>

Use WDK CLI when you want a ready-made local wallet and daemon. Use the [MCP Toolkit](/ai/mcp-toolkit/) when you are building your own MCP server or embedding WDK tools in an application.

## What You Can Do

- Create, import, rename, unlock, lock, and delete named wallets
- Derive addresses, read balances, and query transaction history
- Preview transfers with `--dry-run`, then send native or registered token transfers after confirmation
- Configure networks, tokens, indexer access, and MoonPay on/off-ramp settings
- Run all commands with `--json` for agent or script consumption
- Register `wdk-mcp` with Claude Desktop, Claude Code, or OpenClaw

## Architecture

The CLI and MCP server are thin clients. They do not keep seed phrases or private keys themselves. Wallet operations go through the daemon over local IPC.

```text
wdk CLI      -> local IPC -> wdk-daemon -> WDK wallet modules -> blockchain RPC/indexer
wdk-mcp MCP  -> local IPC -> wdk-daemon -> WDK wallet modules -> blockchain RPC/indexer
```

The daemon starts when a wallet is unlocked. It keeps each unlocked wallet in memory until the wallet is locked, the TTL expires, or the last wallet is locked and the daemon exits.

## CLI vs MCP

| Interface | Use it when | Example |
| --- | --- | --- |
| CLI | Your agent or script can run local shell commands | `wdk get balance --network ethereum --json` |
| MCP | Your AI client supports MCP tools and should call structured wallet actions | `wdk mcp setup --ai-tool claude-desktop` |
| MCP Toolkit | You need to build and customize your own MCP server in code | `new WdkMcpServer(...)` |

The CLI and MCP server share the same daemon and wallet store. You can create and unlock a wallet with `wdk`, then use the MCP tools from your AI client.

## Safety Model

<Callout type="warn">
Use a dedicated development wallet with limited funds while these docs and the CLI module are still in draft. Do not use a production seed phrase to test unmerged code.
</Callout>

Wallet seed files are stored under `~/.config/wdk-cli/wallets/NAME/seed.enc` by default, or under `$XDG_CONFIG_HOME/wdk-cli/wallets/NAME/seed.enc` when `XDG_CONFIG_HOME` is set. If you provide a passphrase, the seed is encrypted before it is written to disk.

Unlocking a wallet sends the passphrase to the daemon over local IPC, initializes WDK wallet instances in memory, and starts a per-wallet TTL. The default TTL is 5 minutes. Use `--ttl 0` only for controlled local development sessions where you accept the longer-lived unlocked wallet.
