# API Reference Automation - First Run Report

**Date:** April 2-3, 2026
**Scope:** `wallet-spark` only (initial rollout)
**PR created:** [#7 - docs(api-ref): auto-update API references](https://github.com/tetherto/wdk-docs-migration/pull/7)

## Overview

The API reference automation pipeline went live on April 2nd. It correctly detected the `wallet-spark` `v1.0.0-beta.12` release tag (published ~16:38 UTC) and auto-generated an updated API reference — zero manual intervention, zero SDK repo changes.

However, the cron fired 3 times overnight and all 3 did identical work. Only the first run was useful; runs 2 and 3 were redundant due to a missing skip-if-already-processed check in the single-module code path.

## Run Results

| Run | Trigger | Timestamp (UTC) | Duration | Useful? | Outcome |
|-----|---------|-----------------|----------|---------|---------|
| 1 | `schedule` | Apr 2, 18:14 | 2m 10s | Yes | Created PR #7 from `v1.0.0-beta.12` |
| 2 | `schedule` | Apr 3, 00:14 | 2m 14s | **No** — redundant | Re-generated same output, updated PR #7 |
| 3 | `schedule` | Apr 3, 06:23 | 2m 12s | **No** — redundant | Re-generated same output, updated PR #7 |

**Root cause of redundant runs:** The single-module path (`--module wallet-spark`) ignores the `--check-updates` flag entirely — it always regenerates unconditionally. Additionally, `last-synced-shas.json` on `main` was never populated (all nulls), so even if the check existed, it would have treated the module as "new" every time. The SHA file gets updated on the `auto/api-ref-update` PR branch, not on `main`, so subsequent runs against `main` never see the updated state.

**Wasted CI:** ~4 minutes of redundant work (2 unnecessary runs x ~2 min each). Not critical, but wasteful at scale when all modules are enabled.

## Issues Found

### Issue 1: Redundant Runs (no skip logic for single modules)

**Problem:** The `--check-updates` flag is only implemented in the `--module all` code path. When the schedule runs `--module wallet-spark --check-updates`, the flag is silently ignored. Every cron cycle regenerates unconditionally, even if the tag hasn't changed since the last run.

**Impact:** 2 wasted CI runs (~4 min total). At scale with all 13 modules, this would mean ~26 min wasted per redundant cycle.

**Fix:** Add tag-check logic to the single-module path: read `last-synced-shas.json`, compare against latest remote tag, skip if unchanged. (PR #8)

### Issue 2: Missing Modules Table in PR Body

**Problem:** The auto-PR body was missing the modules stats table (tag, lines, classes, types). Only the `--module all` code path wrote `.generate-summary.json`, which the workflow reads to build the table.

**Impact:** Reviewers couldn't see at a glance what was generated — no tag version, no line count, no class/type counts.

**Fix:** Single-module runs now also write `.generate-summary.json` with the same structure. (PR #8)

### Issue 3: Noisy Ripple Effect Detection

**Problem:** The ripple check was splitting multi-word section headings (like `### Lightning Fee Estimate`) into individual words and grepping each one. This produced ~30 false positives: `"Lightning"`, `"Options"`, `"Fee"`, `"of"`, `"Spark"`, `"Invoice"` matching in sibling docs as regular English text.

**Impact:** The ripple warnings section was cluttered with noise, making it impossible to spot real issues.

**Fix:** Now reads headings line-by-line and only flags symbols containing a camelCase/PascalCase case transition (`[a-z][A-Z]`). This matches real API identifiers like `getBalance`, `WalletManagerSpark`, `LightningSendRequest` while skipping plain English words. (PR #8)

## Pipeline Architecture

```
Every 6 hours (cron) or manual trigger
         |
         v
  git ls-remote --tags wdk-wallet-spark
         |
         v
  Compare tag+SHA with last-synced-shas.json
         |
    [changed?]---no---> exit (no CI minutes wasted)
         |
        yes
         |
         v
  git clone --depth 1 --branch <tag>
         |
         v
  npm install --ignore-scripts
         |
         v
  TypeDoc -> JSON -> transform.mjs -> MDX
         |
         v
  git diff (check for actual file changes)
         |
         v
  Ripple effect check (removed symbols in sibling docs)
         |
         v
  npm run build (quality gate)
         |
         v
  peter-evans/create-pull-request (create or update PR)
```
